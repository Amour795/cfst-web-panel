用于自动将优选 IP 更新到你的域名下。
对应 -tp：延迟测速端口。一般设为 443，也可选 80, 8443, 2053 等。（默认值: 443）
对应 -n：延迟测速并发线程数。数值越大测速越快，但太大会导致路由器断流或报错。（默认值: 200）
对应 -t：单个 IP 延迟测速次数。次数越多越能反映真实延迟，但会增加总测速时间。（默认值: 4）
对应 -url：用于测试下载速度的文件地址。也可自建测速文件源。
对应 -dt：单个 IP 下载测速最长时间。设置太短容易不准，推荐 5~10 秒。（默认值: 5）
对应 -dn：延迟测速并排序后，从低延迟开始做下载测速的 IP 数量上限。（默认值: 10）
仅在输入栏只有 1 个目标 IP 时生效，对应单目标场景的 -dn，避免单节点重复测速浪费时间。（默认值: 1）
对应 -tl：超过此平均延迟的 IP 将被抛弃，不做下载测速。（默认值: 9999，即不限制）
对应 -tll：低于此平均延迟的 IP 将被抛弃，不做下载测速。（默认值: 0，即不限制）
对应 -tlr：高于此丢包率的 IP 将被抛弃。填 0 会过滤掉任何丢包的节点。（默认值: 1.00，即不限制）
对应 -sl：结果只保留速度高于该值的 IP。如果与 -dn 连用，脚本会尽量寻找满足条件的 IP 直至队列测完。（默认值: 0.00，即不限制）
对应 -dd：开启后跳过下载阶段，-dn/-dt/-sl 均不再生效。结果仅按延迟/丢包排序，大幅缩短测速耗时。
对应 -allip：对提供的所有 IP 进行测速。开启后耗时会更长，但结果覆盖面更广。
对应 -debug：开启后会在服务端后台控制台打印详细的内部调试日志。
对应 -p：测速完成后，前端表格最多保留和显示的结果条数。（默认值: 50）
